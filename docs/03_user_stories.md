# 03. ユーザーストーリー

## 1. このドキュメントの目的

本ドキュメントは AI 寮母 MVP における**ユーザーの体験**をペルソナ別・シナリオ別に定義する。

- 誰が
- どんな状況で
- 何をして
- どんな価値を得るか

を明確にし、`04_functional_spec.md` 以降の実装仕様の根拠とする。

## 2. ペルソナ

### 2.1 学生ペルソナ

#### P-S1: 一人暮らし新入生「山田 春樹」（架空）

- **属性**: 同志社大学 経済学部 1 年生、19 歳、京都市左京区在住
- **出身**: 岡山県
- **状況**: 大学入学と同時に岡山から上京、一人暮らし 3 ヶ月目
- **目標**:
  - 大学生活を充実させたい
  - 何か新しいことに挑戦したい
  - 京都のことをもっと知りたい
- **課題**:
  - 病院や薬局の探し方がわからない
  - ゴミ出しルールが複雑
  - サークルに入りそびれた
  - 周囲の学生が何をしているか見えない
- **LINE 利用**: 日常的に使う、Bot も抵抗なく友だち追加できる

#### P-S2: 目的を探す 2 年生「佐々木 美咲」（架空）

- **属性**: 同志社大学 商学部 2 年生、20 歳、京都市上京区在住
- **出身**: 大阪府
- **状況**: 大学 2 年目、そろそろ「何か始めなきゃ」と焦り始めている
- **目標**:
  - 就活に向けて何か成果を積みたい
  - 地域と関わる活動をしたい
  - 自分の興味の方向性を確かめたい
- **課題**:
  - 興味はあるが具体的に何をすべきかわからない
  - 周りのすごい学生と比べて焦る
  - ボランティアや地域活動の情報が入ってこない

### 2.2 保護者ペルソナ

#### P-P1: 遠方の母「山田 由美」（架空、P-S1 の母）

- **属性**: 47 歳、岡山県在住、パート勤務
- **状況**: 息子（春樹）が京都に進学、初めての一人暮らし
- **目標**:
  - 息子が元気に生活できているか知りたい
  - 過干渉にはなりたくない
  - 息子の自立を尊重したい
- **課題**:
  - 電話しても「大丈夫」しか返ってこない
  - 生活状況がまったく見えない
  - 頑張っていることを共有してほしい
- **LINE 利用**: 家族グループで使う、公式アカウントの友だち追加は経験あり

## 3. ユーザーストーリー（学生）

### 3.1 US-S01: 初回の役割選択

**As a** 一人暮らし新入生の学生
**I want** LINE で AI 寮母を友だち追加した瞬間に、自分が「学生」か「保護者」か選べる
**So that** 迷わず自分向けの機能を使い始められる

**受け入れ基準**:
- [ ] 友だち追加直後にウェルカムメッセージが届く
- [ ] Quick Reply で「👨‍🎓 学生です」「👨‍👩‍👧 保護者です」を選べる
- [ ] 学生を選択すると学生向けのオンボーディングが始まる
- [ ] 誤って選択した場合、後から役割を変更するコマンドが案内される

**関連機能**: FR-S1, FR-P1, FR-C2

### 3.2 US-S02: プロフィール登録

**As a** 学生
**I want** 対話形式で自分の情報（大学・学部・興味関心等）を登録できる
**So that** AI 寮母が自分に合った提案をしてくれる

**受け入れ基準**:
- [ ] 学生選択後、プロフィール登録の案内が届く
- [ ] 以下の項目を対話形式で聞かれる:
  - 大学名
  - 学部
  - 学年
  - 興味関心（複数選択可、Quick Reply）
  - 最近頑張っていること（自由記述）
  - やってみたいこと（自由記述）
- [ ] 入力途中でキャンセルできる
- [ ] 登録完了後、確認メッセージが届く
- [ ] 未登録状態で他機能を使おうとするとプロフィール登録を促される

**関連機能**: FR-S2

### 3.3 US-S03: やりたいこと相談（主軸機能）

**As a** 目的を探す学生
**I want** 「何かやりたい」と Bot に相談したときに、具体的な活動候補を提案してもらう
**So that** 行動のきっかけをつかめる

**受け入れ基準**:
- [ ] リッチメニューまたは Quick Reply で「やりたいこと相談」を選べる
- [ ] プロフィールと地域データを元に、AI が 2〜3 件の活動を提案する
- [ ] 提案は Flex Message のカルーセルで表示される
- [ ] 各カードに以下が含まれる:
  - 活動名
  - 概要（1〜2 文）
  - 場所・日時（該当あれば）
  - 「参加した」ボタン
  - 「詳しく聞く」ボタン
- [ ] 「他の案も見る」ボタンで再提案を受けられる
- [ ] 「参加した」を押すと経験投稿フローに誘導される

**関連機能**: FR-S4, FR-S6

### 3.4 US-S04: 生活相談

**As a** 一人暮らし新入生
**I want** 体調不良やゴミ出しなど生活の困りごとを Bot に相談できる
**So that** 24 時間いつでも地域特化の助けが得られる

**受け入れ基準**:
- [ ] テキストで自由に相談を送れる
- [ ] AI が地域情報と先輩投稿を参照した回答を返す
- [ ] 医療的な内容の場合、「症状が続くなら医療機関を受診してください」と誘導する
- [ ] 緊急性が高い内容の場合、110/119 の案内をする
- [ ] 参照した先輩投稿がある場合、「先輩の体験より」といった注記を付ける
- [ ] 回答は 300 文字以内を目安とする（長くなる場合は Flex Message で構造化）

**関連機能**: FR-S5

### 3.5 US-S05: 経験投稿

**As a** 学生
**I want** 参加したイベントや利用した店舗の情報を Bot に投稿できる
**So that** 後輩の役に立てて、自分の頑張りも記録される

**受け入れ基準**:
- [ ] リッチメニューまたは Quick Reply から「経験を投稿」を選べる
- [ ] カテゴリ（イベント / 店舗 / 生活の知恵 / その他）を Quick Reply で選ぶ
- [ ] タイトル、本文、対象地域（任意）を対話形式で入力する
- [ ] 「頑張ったこととして保護者に共有する？」を確認される
- [ ] 投稿完了時、確認メッセージが届く
- [ ] 投稿は JSON に永続化される

**関連機能**: FR-S6, FR-S8

### 3.6 US-S06: 保護者への招待コード発行

**As a** 学生
**I want** 保護者に AI 寮母を紹介するための招待コードを発行できる
**So that** 保護者に自分の頑張りを共有できる

**受け入れ基準**:
- [ ] 「保護者連携」メニューから招待コード発行ができる
- [ ] 6 桁の英数字コードが発行される
- [ ] コードには有効期限（例: 24 時間）がある
- [ ] コードは 1 回使うと無効化される
- [ ] 共有用の案内テキスト（コード + 手順）が生成される
- [ ] 共有内容は「保護者への共有はいつでも解除できます」と明記される

**関連機能**: FR-S7

## 4. ユーザーストーリー（保護者）

### 4.1 US-P01: 初回の役割選択

**As a** 遠方に住む学生の母親
**I want** 友だち追加時に「保護者」を選べる
**So that** 迷わず保護者向けの機能に進める

**受け入れ基準**:
- [ ] 学生と同じウェルカムメッセージで役割選択を求められる
- [ ] 「保護者です」を選択すると保護者向けのオンボーディングが始まる

**関連機能**: FR-P1

### 4.2 US-P02: 学生との連携

**As a** 保護者
**I want** 子どもから受け取った招待コードを入力して連携できる
**So that** 子どもの頑張りを見守れる

**受け入れ基準**:
- [ ] 保護者選択後、「学生から受け取った招待コードを教えてください」と案内される
- [ ] 6 桁のコードを入力すると連携完了メッセージが届く
- [ ] コードが無効・期限切れの場合はエラーメッセージと再入力の案内
- [ ] 連携完了後、「今月のレポート」ボタンが Quick Reply に表示される
- [ ] 学生本人には「保護者と連携しました」と通知される

**関連機能**: FR-P2

### 4.3 US-P03: 月次サマリー閲覧

**As a** 保護者
**I want** 「今月のレポート」ボタンを押すと子どもの頑張ったことが見られる
**So that** 過干渉にならず、適度な距離感で見守れる

**受け入れ基準**:
- [ ] Quick Reply またはリッチメニューから「今月のレポート」を選べる
- [ ] 連携している学生の当月の「頑張ったこと」が Flex Message で表示される
- [ ] 該当データがない場合は「今月はまだ記録がありません」と表示される
- [ ] 表示内容は学生本人が「頑張ったこととして共有」を選んだ投稿のみ
- [ ] 会話ログや生活相談内容は共有されない
- [ ] 直近 3 ヶ月分まで遡れる（余裕があれば）

**関連機能**: FR-P3

## 5. デモシナリオでの物語構成

`08_demo_scenario.md` で詳述するが、上記ストーリーを組み合わせて以下の物語を作る。

### 5.1 メインストーリー

1. **US-S01** 学生役アカウントで友だち追加、役割選択
2. **US-S02** プロフィール登録（P-S1 春樹の設定を使う）
3. **US-S03** やりたいこと相談 → AI が地域活動を提案
4. **US-S06** 保護者連携用の招待コード発行
5. **US-P01** 保護者役アカウントで友だち追加、役割選択
6. **US-P02** 招待コードで連携
7. **US-S05** 学生役で「参加した」→ 経験投稿（頑張ったこととして共有）
8. **US-P03** 保護者役で「今月のレポート」→ 学生の頑張り確認
9. **US-S04** 学生役で生活相談（「体調悪い、近くの病院」）→ 地域情報活用回答

### 5.2 補足で見せる価値

- 先輩投稿を活用した回答（SECI モデル）
- Flex Message の視覚的な提案 UI
- 保護者への「監視ではない見守り」の設計

## 6. スコープ外のユーザーストーリー

以下は本 MVP では実装しない。発表資料の「今後の展望」で言及する可能性はある。

- 学生同士のマッチング（自分と似た興味の学生を紹介）
- 地域店舗からの求人・イベント募集
- 保護者への緊急通知
- クーポンによる地域店舗送客
- 就活記録への活用
- モデレーション対応（不適切投稿の通報）
