# AI寮母 完全版コンテキストドキュメント

## 0. このドキュメントの目的

このドキュメントは、ビジネスアイデア大会で提案するプロダクト「AI寮母」について、企画・発表・開発に必要な情報を1つに集約した完全版コンテキストである。

今後、AIや開発メンバーにプロダクトの背景・課題・機能・提供価値・ビジネスモデルを説明する際に、複数のファイルを参照させなくても、このファイル単体で全体像を理解できることを目的とする。

このファイルでは、書類選考時点の内容と、決勝に向けて議論された新しい方向性の両方を統合している。ただし、現在の方針としては、より新しい「決勝に向けた整理」を優先する。

なお、書類選考用エントリーシートに含まれていた電話番号やメールアドレスなどの個人連絡先は、AI開発用コンテキストとしては不要であり、共有リスクが高いため、この統合版には記載しない。

## 2. プロダクトの中心コンセプト

### 2.1 一文での説明

AI寮母は、学生マンションに埋もれている学生の経験・地域情報・行動機会をAIで形式知化し、初めての一人暮らしの不安を減らしながら、学生が地域や他者とつながり、学び・成長するきっかけを生み出すサービスである。

### 2.2 書類選考時点のコンセプト

書類選考時点では、AI寮母は以下のように説明されていた。

初めて一人暮らしをする学生と、その学生を送り出す保護者の不安に着目する。AI寮母は、チャットを通して先輩入居者が蓄積した生活の知見を学習し、現在の入居者に対して、不安を解消し、日常を豊かにする情報を提供する。

株式会社ジェイ・エス・ビーが掲げる「学び・成長・つながり」を、以下の形で実現する。

- 学び：先輩たちが得た生活知をAIに蓄積し共有する。
- 成長：日常の課題を自ら解決し、知識を組織資産化する力を育む。
- つながり：入居者同士や地域社会とのつながりをAIが仲介する。

さらに、保護者にはプライバシーに配慮した子どもの生活状況レポートを提供することで、学生の自立を尊重しながら安心感を届ける。

この段階では、AI寮母は学生マンションを単なる住居ではなく「知識が受け継がれる場所」へ進化させる新しい価値提案として位置付けられていた。

### 2.3 決勝に向けた新しい方向性

決勝に向けた議論では、単なる生活支援AIではなく、「何かやりたいけど、何をやればよいかわからない学生」に対して、学び・成長・つながりのきっかけづくりを行うサービスとして再整理されている。

中心にある考え方は以下である。

- JSBは、社会課題解決ができる若者を育成したい。
- 学生は「何かやりたい」「成長したい」と思っているが、具体的に何をすればよいかわからないことがある。
- 地域社会も、学生と何かをしたい、学生に地域活動や困りごと解決に関わってほしいというニーズを持っている。
- しかし、学生側の興味関心や目標、地域側のディープな情報やニーズは、それぞれ暗黙知として埋もれている。
- その結果、学生と地域が交わるきっかけが生まれていない。
- AI寮母は、学生情報と地域情報を形式知化し、学生と地域、学生同士をつなぐ。

このため、最新版の方向性では、生活の困りごとの解決だけでなく、学生の学び・成長・つながりを促す「きっかけづくり」が重要な軸になっている。

### 2.4 現時点での位置付け

現時点のAI寮母は、以下の2つの側面を併せ持つプロダクトとして理解できる。

1. 生活支援AI
   - 初めての一人暮らしの不安を解消する。
   - 地域やマンションに特化した生活情報を提供する。
   - 保護者に安心感を届ける。

2. 成長機会創出AI
   - 学生の興味・関心・目標に応じて、地域イベントや活動機会を提案する。
   - 地域の困りごとや参加募集を、求人募集のような形で学生に届ける。
   - 学生同士や地域とのつながりを生み出す。

ただし、7月2日時点では「機能が多くてブレすぎているのではないか」「機能を減らした方がよいのではないか」という課題意識も出ている。そのため、今後の開発・発表では、どの機能をMVPとして優先するかを明確にする必要がある。

## 3. 背景・問題意識

### 3.1 JSB側の背景

JSBは「豊かな生活空間の創造」という経営理念の根底に、社会課題を解決できる若者を育成したいという目的を持っていると整理されている。

学生マンションは、これまで設備や立地などのハード面で価値を提供してきた。一方で、学生の学び・成長・つながりを促すソフト面のアプローチには伸びしろがある。

AI寮母は、学生マンションを単なる居住空間ではなく、学生が地域と関わり、学び、成長するインフラへ進化させる提案である。

### 3.2 学生側の背景

学生側には、主に以下の背景がある。

- 初めての一人暮らしでは、生活面の不安が大きい。
- 夜間の急病、病院や薬局の探し方、行政手続き、地域特有のゴミ出しルールなど、生活インフラに関する基礎的なつまずきがストレスになる。
- 周辺に住んでいても、地域の飲食店、イベント、生活に役立つ施設などを十分に把握できていない。
- 同じマンションに住んでいても、隣人の顔や専攻すら知らないことが多い。
- 何かやりたい、成長したいという意欲はあっても、具体的に何をすればよいかわからず、行動のきっかけがない。

### 3.3 保護者側の背景

保護者側には、主に以下の背景がある。

- 遠方で暮らす子どもの生活状況が見えにくい。
- 「ちゃんと生活できているか」という心配が続く。
- 過度に干渉したいわけではないが、最低限の安心材料がほしい。
- 子どもの自立を尊重しながら、緊急時にはすぐに気づける仕組みがあると安心できる。

書類選考時点では、保護者の不安を解消することが大きな価値として位置付けられていた。

### 3.4 地域側の背景

地域側には、主に以下の背景がある。

- 地方自治体や地域社会は、地方創生や伝統文化の継承のために、学生や若者と関わりたいというニーズを持っている。
- 地域店舗や地域イベントの運営者は、学生に来てほしい、参加してほしい、地域の困りごとを手伝ってほしいと考えている可能性がある。
- しかし、地域側から学生へアプローチする接点が少ない。
- 結果として、学生の若い活力が地域に十分接続されていない。

### 3.5 情報の暗黙知化

AI寮母の重要な問題意識は、学生マンションや地域に存在する有益な情報が、暗黙知として埋もれていることである。

具体的には、以下のような情報が、個人の経験や一部の学生の知識にとどまり、次の学生へ継承されていない。

- 安くて学生に優しい飲食店。
- 病気になったときに利用しやすい病院。
- 夜間や休日に頼れる薬局。
- 地域特有のゴミ出しルール。
- 行政手続きのコツ。
- 地域イベントの情報。
- 生活の知恵。
- サークル、バイト、地域活動などの経験。
- 学生が実際に行動して得た学び。

このような情報が共有・蓄積されないため、毎年新しい入居者が同じ不安や困りごとを繰り返している。

## 4. 解決したい課題

### 4.1 本質的課題

AI寮母が解決したい本質的課題は、以下のように整理できる。

学生マンションには、先輩入居者の経験、地域情報、生活上の知恵、学生の興味関心や目標、地域側のニーズが存在しているにもかかわらず、それらが組織的に共有・継承されず、学生の不安解消や学び・成長・つながりの機会に十分活用されていない。

### 4.2 課題1：初めての一人暮らしに伴う生活の不安

初めて一人暮らしをする学生は、生活上のさまざまな不安を抱える。

具体例は以下である。

- 体調不良時にどの病院へ行けばよいかわからない。
- 夜間や休日に薬局・病院を探す必要がある。
- 地域特有のゴミ出しルールがわからない。
- 行政手続きの方法がわからない。
- 生活費管理、家事、アルバイト、周辺環境の変化に戸惑う。
- 周辺の生活インフラや地域情報がわからない。

書類選考時点では、株式会社CHINTAIの「お部屋探しに関する不安や情報収集の役割分担についてのアンケート調査」（リセマム掲載）が参考資料として使われていた。

その図では、子どもが引越しをした後の生活面の不安として、以下の項目が示されていた。

| 不安項目 | 割合 |
|---|---:|
| 体調管理 | 59.8% |
| 生活習慣の乱れ | 50.8% |
| 生活費の管理 | 38.0% |
| 入居関係 | 37.3% |
| 家事 | 34.8% |
| アルバイト | 31.5% |
| 周辺の環境の変化 | 22.5% |
| 就職活動 | 11.8% |
| 親子間での定期的な連絡 | 11.5% |
| 不安はない | 8.3% |
| その他 | 0.5% |

このデータは、保護者が子どもの一人暮らしに対して、生活面・体調面の不安を抱えていることを示す参考材料として用いられている。

### 4.3 課題2：やりたいことが見つからない学生の課題

決勝に向けた議論で新たに重要視されているのが、「何かやりたいけど、何をやればよいかわからない学生」の課題である。

学生は、以下のような状態に陥ることがある。

- 何か新しいことをしたい。
- 成長したい。
- 地域や人とつながりたい。
- しかし、具体的に何をすればよいかわからない。
- 自分に合う活動やイベントを探す方法がない。
- 周囲の学生が何をしているか見えない。
- 行動を起こすきっかけがない。

AI寮母は、学生の興味・関心・目標・最近頑張っていることなどを把握し、その学生に合った地域イベント、活動機会、ボランティア、店舗、学びの機会を提案することで、この課題を解決しようとしている。

### 4.4 課題3：地域との断絶と情報の暗黙知化

学生は地域に住んでいるにもかかわらず、地域の魅力や生活に役立つ情報を十分に把握できていない。

書類選考時点では、チーム内6人中4人が「近隣の学生向け飲食店、地域イベント、生活に役立つ施設などを十分に把握できていない」と回答したとされている。

このことから、学生は地域に住んでいながらも、日常生活に役立つローカル情報へ十分にアクセスできていない実態があると推測されている。

また、安くて学生に優しい飲食店や生活の知恵は一部の学生だけの暗黙知にとどまり、毎年入れ替わる新入生へ継続的に共有される仕組みがない。

### 4.5 課題4：学生同士の分断

同じ建物に住んでいるにもかかわらず、学生同士の交流は希薄である。

書類選考時点では、マンションに住むメンバー4人が「隣人との交流がない」と回答したとされている。

このことから、マンションに住む住人同士のつながりは希薄であると推測されている。

物理的には近くにいるにもかかわらず、互いの専攻、サークル、趣味、得意分野、困っていることを知らない。そのため、教え合い、助け合い、共創する機会が失われている。

### 4.6 課題5：JSBの学生支援を収益性・物件価値へつなげる課題

JSB側には、学生支援や成長支援の取り組みを、単なる善意やボランティアで終わらせず、事業として成立させたいという課題がある。

就活支援、エントリーシート支援、企業説明会のような取り組みは行われているが、それ自体が大きな収益源になりにくい可能性がある。

AI寮母は、学生の成長に役立つことと、JSBの収益・物件価値向上につながることを両立させるサービスとして構想されている。

## 5. ターゲット・ステークホルダー

### 5.1 学生マンション入居者

主な利用者である。

対象となる学生像は以下である。

- 初めて一人暮らしをする学生。
- 生活面で不安を抱える学生。
- 地域情報をあまり知らない学生。
- 同じマンションの住人とつながりがない学生。
- 何かやりたいが、何をすればよいかわからない学生。
- 学び・成長・つながりの機会を求めている学生。

### 5.2 保護者

学生を送り出す保護者も重要な顧客・利用者である。

保護者は、子どもの自立を尊重しつつも、生活状況が完全に見えないことに不安を抱えている。

AI寮母は、プライバシーに配慮した安心レポートや緊急時通知を通じて、保護者に適度な安心感を提供する。

### 5.3 JSB

JSBにとって、AI寮母は以下の価値を持つ。

- 学生マンションの差別化。
- 物件価値向上。
- 入居率向上。
- 顧客満足度向上。
- 管理業務の効率化。
- 学生マンション独自の知識資産の蓄積。
- 学生支援を収益やブランド価値につなげる仕組み。

### 5.4 地域店舗・地域社会

地域店舗や地域社会は、AI寮母を通じて学生と接続できる。

主な価値は以下である。

- 地域店舗への送客。
- クーポンやポイントによる来店促進。
- 地域イベントへの学生参加。
- 地域の困りごと解決への学生参加。
- 地域コミュニティの活性化。
- 若者の視点や活力の取り込み。

### 5.5 地方自治体・地域団体

決勝に向けた整理では、地方自治体や地域社会が、地方創生や伝統文化の継承のために学生と関わりたいという背景も示されている。

そのため、自治体や地域団体も将来的なステークホルダーになり得る。

## 6. サービス内容

### 6.1 基本機能

AI寮母は、学生マンション専用の生活支援・成長支援AIチャットである。

利用者はAIに対して、以下のような相談ができる。

- 生活の困りごと。
- 地域の気になる情報。
- 病院や薬局の探し方。
- 行政手続き。
- ゴミ出しルール。
- 安くておいしい飲食店。
- 地域イベント。
- サークルやバイトの情報。
- 自分に合った活動機会。
- 最近頑張っていることや目標に関連する行動の提案。

一般的な生成AIとの違いは、AI寮母がその地域・そのマンションに特化した情報を持つ点である。

### 6.2 AIが扱う情報

AI寮母は、以下の情報を活用する。

1. 学生の情報。
2. 先輩入居者の経験。
3. 地域の情報。
4. マンション固有の情報。
5. 学生の投稿内容。
6. 学生とAIの会話ログ。
7. 地域店舗や地域団体からの情報。
8. 学生の近況報告。
9. 保護者向けレポートに使う範囲の利用状況・活動状況。

ただし、どこまでパーソナルな情報を扱うかは、弁護士などの専門家に確認する必要があるとされている。

### 6.3 学生情報として想定されている項目

デモ版や本番版で扱う可能性のある学生情報は以下である。

- 大学。
- 学部。
- サークル。
- バイト。
- 趣味。
- 目標。
- やりたいこと。
- 興味がある分野。
- 最近頑張っていること。
- 出身地。
- 登録時アンケートの回答。
- 会話ログ。
- 近況報告。

デモ版を作る際は、学生の個人情報（プロフィール・投稿・会話ログ）は実在のものを使わず、架空の学生情報を使う方針である（一方、地域情報・イベント・店舗は §6.4 で述べる通り実在情報ベース）。

### 6.4 地域情報として想定されている項目

AI寮母が扱う地域情報は以下である。

- 行政の情報。
- 地域イベント。
- 飲食店。
- お店。
- 地域の文化。
- 地域の歴史。
- 地域の困りごと。
- 地域活動の参加募集。
- ボランティア情報。
- 学生向けサービス。
- クーポン対象店舗。

地域情報は、手動で入力していくことが想定されている。

地域店舗や地域団体側から登録・宣伝してもらう形も検討されているが、MVP では対象外。

MVP デモでは、**同志社大学今出川キャンパス周辺の実在情報**（大学施設・図書館・行政窓口・保健センター・実在店舗・大学公式イベント等）を手動収集して seed に投入する方針である。実在の民間店舗については、営業状況変化のリスクに配慮して「情報が古い可能性がある」旨の鮮度注記を必ず添える（詳細は `.codex/rules/project_rules.md` の「データ運用」節参照）。

### 6.5 先輩入居者の経験として想定されている情報

AI寮母では、先輩入居者が実際に経験した情報をAIに蓄積する。

想定されている投稿内容は以下である。

- 病気になった際に利用した病院。
- 行政手続きの経験。
- 安くておいしい飲食店。
- 地域イベントへの参加経験。
- 生活に役立つ施設。
- マンション独自のルール。
- 手続きのコツ。
- 地域での活動経験。
- 生活上の困りごとと解決方法。

これにより、毎年失われていた生活知や地域知を、次の入居者が活用できるようにする。

### 6.6 学生と地域をつなぐ機能

決勝に向けた方向性では、AI寮母が学生と地域をつなぐ役割を持つ。

具体的には、以下のような機能が想定されている。

- 地域の情報をAIが発信する。
- 地域イベントへの参加を促す。
- 地域の困りごとを学生に紹介する。
- 地域活動への参加者を、求人募集のような形で募る。
- 学生の興味関心に合わせて、地域の活動機会を提案する。
- クーポンを付与することで、実際の行動につなげる。

この機能は、「受け身の生活情報提供」にとどまらず、学生に学びの機会を提供するために重要である。

### 6.7 学生同士をつなぐ機能

AI寮母は、学生同士のつながりも生み出す可能性がある。

想定される機能は以下である。

- 他の学生が何をしているかをAI寮母が回答する。
- 同じ興味関心を持つ学生を見つける。
- 同じマンション内の経験や知恵を共有する。
- 困りごとを持つ学生と、経験者をつなぐ。
- 直接個人を特定しない範囲で、周囲の学生の活動傾向を伝える。

ただし、学生同士の接続にはプライバシー上の配慮が必要である。どこまで個人情報を出すか、本人同意をどう取るかは要検討である。

### 6.8 保護者向け安心レポート

書類選考時点から、保護者向け機能として「プライバシーに配慮した生活状況レポート」が検討されている。

想定される内容は以下である。

- AIの利用頻度。
- 日常の挨拶ログ。
- 月ごとに学生が頑張ったこと。
- 生活状況の大まかな傾向。
- 緊急時の通知。
- 急病時やトラブル時のセーフティネット。

決勝に向けた議論では、「毎月頑張ったことを書いて、保護者に送る」「その情報が就活のときに役立つかもしれない」という案も出ている。

ただし、この機能は監視に見えないようにする必要がある。学生の自立を尊重し、本人の同意を前提に、保護者へ共有する内容を限定する必要がある。

### 6.9 クーポン・ポイント機能

学生が情報を共有したり、地域活動に参加したりするインセンティブとして、地域店舗で使えるポイントやクーポンを付与する案がある。

想定される流れは以下である。

1. 学生が生活情報や地域情報を共有する。
2. 情報提供に対してポイントやクーポンを付与する。
3. 学生が地域店舗でポイントやクーポンを利用する。
4. 地域消費が生まれる。
5. 地域店舗の売上や地域活性化につながる。
6. 学生の情報共有がさらに促進される。

この仕組みによって、知識共有、ポイント獲得、地域消費、地域活性化の循環を作ることが目指されている。

## 7. サービスフロー

### 7.1 基本的な情報蓄積フロー

AI寮母の基本的なサービスフローは以下である。

1. 学生がプロフィールや興味関心を登録する。
2. 学生が生活経験や地域情報を投稿する。
3. 地域店舗や地域団体が地域情報・イベント情報・参加募集を登録する。
4. 投稿内容を確認・整理する。
5. AIがマンション・地域ごとの知識として蓄積する。
6. 新入居者や現入居者がチャットで相談する。
7. AIがその地域・そのマンションに特化した回答を生成する。
8. AIが必要に応じて、地域活動や学びの機会を提案する。
9. 学生が行動する。
10. 行動結果や体験が再び投稿され、知識として蓄積される。

### 7.2 「投稿 → 確認 → 蓄積 → AIが参照」の流れ

AI寮母の差別化を説明する際には、以下の流れが重要である。

投稿 → 確認 → 蓄積 → AIが参照

一般的なAIチャットは、広く一般的な情報には答えられるが、そのマンション・その地域・その学生集団に固有の暗黙知には答えにくい。

AI寮母は、入居者自身が情報提供者となり、情報を投稿し、AIがそれを参照することで、マンション固有の知識資産を作っていく。

### 7.3 学生側からの利用フロー

学生側の利用フローは以下である。

1. 学生がAI寮母に相談する。
2. AI寮母が学生のプロフィール、現在地、居住マンション、地域情報、過去の投稿を参照する。
3. AI寮母が地域特化・マンション特化の回答を返す。
4. 必要に応じて、活動機会やイベントを提案する。
5. 学生が行動する。
6. 行動後、感想や経験を近況報告・投稿として入力する。
7. その情報が次の学生に活用される。

### 7.4 地域側からの利用フロー

地域側の利用フローは以下である。

1. 地域店舗、自治体、地域団体が情報を登録する。
2. イベント、困りごと、参加募集、クーポンなどをAI寮母に掲載する。
3. AI寮母が学生の興味関心に合わせて情報を提案する。
4. 学生が地域店舗やイベントに参加する。
5. 地域側は学生との接点を得る。
6. 地域活性化や送客につながる。

### 7.5 保護者側の利用フロー

保護者側の利用フローは以下である。

1. 学生本人の同意のもと、保護者向け安心レポートを設定する。
2. AI寮母が学生の利用状況や近況報告を、プライバシーに配慮した形で整理する。
3. 月次などのタイミングで保護者へ共有する。
4. 緊急時には、必要に応じて通知する。
5. 保護者は過度に干渉せず、適度な距離感で安心を得る。

## 8. SECIモデルによる知識継承

AI寮母の理論的な柱として、SECIモデルを使うことができる。

SECIモデルは、暗黙知と形式知の変換を通じて、組織的に知識を創造・共有していく考え方である。

AI寮母では、学生マンション内にある生活知や地域知を、以下のサイクルで蓄積・継承する。

### 8.1 S：共同化（Socialization）

同じマンションに住む仲間や先輩の横顔を見て、悩みや経験を共感し合う段階である。

AI寮母の文脈では、学生同士が同じマンションや地域で生活し、似た困りごとや経験を持つことが出発点になる。

### 8.2 E：表出化（Externalization）

自分が得た口コミやトラブル経験を文字・データにしてAI寮母に投稿する段階である。

具体例は以下である。

- 病院の口コミ。
- 手続きのコツ。
- おすすめ店舗。
- 地域イベント。
- 生活の困りごとと解決方法。

### 8.3 C：連結化（Combination）

AIが膨大な住民の口コミを整理・構造化し、その地域特有の知恵のデータベースへ昇華させる段階である。

AI寮母は、投稿情報・地域情報・学生情報を組み合わせて、検索・回答・提案に使える知識として整理する。

### 8.4 I：内面化（Internalization）

後輩たちがAIから生きた知恵を学び、自分自身の生活アビリティやスキルとして吸収し、自立する段階である。

この循環によって、退去や卒業によって失われていた生活知が、学生マンションの資産として残り続ける。

## 9. 提供価値

### 9.1 学生への提供価値

学生への主な価値は以下である。

#### 9.1.1 生活不安の解消

初めての一人暮らしにおける不安を軽減する。

- 24時間365日の即時サポート。
- 夜間や休日でも体調不良時の病院・薬局を相談できる。
- 行政手続きの方法を相談できる。
- ゴミ出しルールなどの地域特有の情報を確認できる。
- マンション独自のルールを確認できる。

#### 9.1.2 ローカル情報へのアクセス

その地域、そのマンションに住む学生だからこそ知っている情報を得られる。

- 安くておいしい飲食店。
- 学生に優しい店舗。
- 生活に役立つ施設。
- 地域イベント。
- 地域の文化や歴史。

#### 9.1.3 学び・成長のきっかけ

「何かやりたいが、何をすればよいかわからない」状態から脱却できる。

- 自分の興味に合う地域活動を知る。
- イベントやボランティアに参加する。
- 地域の困りごと解決に関わる。
- 他の学生の活動を知る。
- 自分の頑張りを記録できる。

#### 9.1.4 つながりの創出

同じマンションに住む学生や地域社会とつながるきっかけを得られる。

- 同じ興味を持つ学生を知る。
- 地域イベントに参加する。
- 店舗や地域団体との接点を持つ。
- 学生の経験が次の学生に受け継がれる。

### 9.2 保護者への提供価値

保護者への主な価値は以下である。

#### 9.2.1 適度な距離感での見守り

学生の自立を妨げず、過度な干渉を避けながら、生活状況を把握できる。

#### 9.2.2 安心感の提供

子どもがAI寮母を活用して生活上の困りごとを解決できること自体が、保護者の安心につながる。

#### 9.2.3 緊急時のセーフティネット

急病時や万が一のトラブルの際に通知が届く仕組みにより、遠方で見守る不安を軽減できる。

#### 9.2.4 成長の可視化

学生が毎月頑張ったことや取り組んだ活動を、本人の同意のもとで共有できれば、保護者にとって子どもの成長を感じられる材料になる。

### 9.3 JSBへの提供価値

JSBへの主な価値は以下である。

#### 9.3.1 物件の差別化

「AI寮母が見守るマンション」「学び・成長・つながりを生むマンション」という独自のコンセプトにより、一般賃貸や他社物件との差別化ができる。

#### 9.3.2 入居率・顧客満足度の向上

学生本人だけでなく、実質的な意思決定に影響を持つ保護者の安心感を高めることで、選ばれ続ける物件になる可能性がある。

#### 9.3.3 管理業務の効率化

設備トラブル、ゴミ出し、地域情報などの日常的な問い合わせにAIが一次対応することで、スタッフの業務負荷を軽減できる。

#### 9.3.4 知識資産の蓄積

学生マンションごとの生活知や地域知をデータベース化することで、他社が模倣しにくい知識資産を構築できる。

#### 9.3.5 学生支援と収益性の両立

学生の成長や地域接続に役立つソフト面の支援を、物件価値の向上やサブスク・広告収益につなげられる可能性がある。

### 9.4 地域社会への提供価値

地域社会への主な価値は以下である。

#### 9.4.1 地域店舗への送客

AIを通じてローカル情報やクーポンを提供することで、マンション内の学生を地域の飲食店や商店へ送客できる。

#### 9.4.2 地域コミュニティの活性化

地域イベントやボランティア活動への参加を促すことで、学生が地域社会の活力となるきっかけを作る。

#### 9.4.3 地域課題解決への学生参加

地域の困りごとや活動募集を学生に届けることで、若者の力を地域課題解決に活かすことができる。

#### 9.4.4 地方創生・文化継承への貢献

自治体や地域団体が持つ「学生と関わりたい」というニーズに応え、地域文化や伝統の継承、地域活性化に貢献できる可能性がある。

## 10. ビジネスモデル

### 10.1 収益源の全体像

書類選考時点で提示されていた主な収益源は以下である。

1. 保護者向け安心サブスク。
2. 地域店舗からの広告・掲載収益。
3. 地域店舗で使えるポイント・クーポンによる送客モデル。
4. 間接的な入居率向上・物件価値向上。
5. 管理業務効率化によるコスト削減。

### 10.2 保護者向け安心サブスク

保護者向けに、プライバシーに配慮した安心レポートや緊急時通知を提供する月額課金モデルが想定されている。

書類選考時点では、以下の試算が示されていた。

- 月額：500円。
- 想定利用者数：30,000人。
- 年間売上：1億8,000万円。

計算式は以下である。

500円 × 30,000人 × 12か月 = 180,000,000円 = 1億8,000万円

### 10.3 地域店舗広告収益

地域店舗から掲載料・広告料を得るモデルである。

書類選考時点では、以下の試算が示されていた。

- 月額：2,000円。
- 想定利用店舗数：2,000店舗。
- 年間売上：4,800万円。

計算式は以下である。

2,000円 × 2,000店舗 × 12か月 = 48,000,000円 = 4,800万円

### 10.4 合計売上試算

上記2つを合計すると、書類選考時点の売上例は以下である。

1億8,000万円 + 4,800万円 = 2億2,800万円 / 年

この数字は、確定した事業計画ではなく、ビジネスモデルの可能性を示すための試算である。

### 10.5 地域貢献インセンティブ型モデル

学生に地域店舗で使えるポイントやクーポンを付与することで、以下の循環を作る。

知識共有 → ポイント獲得 → 地域消費 → 地域活性化

このモデルの狙いは以下である。

- 学生がAIに情報を蓄積するインセンティブを生む。
- 地域店舗に学生を送客する。
- 地域店舗側に広告出稿・掲載料支払いの理由を作る。
- 学生・地域・JSBの三者に価値を出す。

### 10.6 間接的な経済効果

書類選考時点では、少子化が進む一方で大学進学率は上昇しており、国内には約180万人の学生マーケットが存在すると整理されていた。

また、そのうち約80%、つまり約144万人は一般賃貸住宅を選択しているとされていた。

AI寮母は「保護者の安心」という明確な価値を提供することで、一般賃貸との差別化を実現する提案である。

仮に一般賃貸利用者の数%をUniLifeへ誘引できた場合でも、家賃収入ベースで数十億円規模の経済効果が期待できる、という説明が書類選考時点で提示されていた。

ただし、この市場規模や経済効果の数字は、決勝に向けて出典や計算根拠を再確認する必要がある。

## 12. MVP・デモに関する整理

### 12.1 現時点の課題意識

決勝に向けた議論では、機能が多く、プロダクトの焦点がブレているという問題意識が出ている。

特に、以下の機能が同時に存在している。

- 生活支援AI。
- 保護者向け安心レポート。
- 地域情報提供。
- 地域活動マッチング。
- 学生同士のつながり。
- クーポン・ポイント。
- 学生活動の記録。
- 就活への活用。

そのため、MVPでは機能を絞る必要がある。

### 12.2 7月2日時点で議論されている軸

7月2日時点では、以下の方向性が議論されている。

- AIチャットによって地域に特化した情報を得ることができる。
- ただし、それだけでは受け身であり、学びが弱い。
- AI側から学びの機会を提供する必要がある。
- 保護者の不安解消も継続する。
- 学生が地域の情報を入力し、先輩の情報が引き継がれることで「つながり」を示す。
- 学生のやりたいことが見つからない課題も入れる。
- 毎月頑張ったことを書いて保護者に送る案もある。

### 12.3 デモ版で使う情報

デモ版では、データ種別ごとに扱いを分ける。

**架空（プライバシー最優先）**

- 学生プロフィール（3〜5 名分）
- 学生の投稿・会話ログ・近況報告
- 先輩入居者の経験投稿（20 件目安）
- 保護者アカウント

学生情報の例として、以下の項目を架空で用意する。大学・学部・サークル・バイト・趣味・目標・やりたいこと・興味分野・最近頑張っていること・出身地。

**実在情報ベース（説得力最優先）**

- 地域情報 30 件（大学施設・図書館・保健センター・行政窓口 等の公開情報）
- イベント 10 件（大学公式・行政主催イベント。日付は「毎年◯月開催」等で古風化回避）
- 店舗 15 件（民間実名店舗。営業状況変化リスクに配慮し「情報鮮度注記」を全レコード必須）

地域情報は、**同志社大学今出川キャンパス周辺**（京都市上京区・中京区・北区）に集中させる。詳細ポリシーは `.codex/rules/project_rules.md` の「データ運用」節を参照。

### 12.4 デモで見せるべき体験の候補

デモで見せる体験として、以下が考えられる。

1. 学生がAI寮母に生活相談をする。
   - 例：体調が悪い、近くの病院を知りたい。
   - 例：ゴミ出しルールを知りたい。
   - 例：安く食べられる店を知りたい。

2. 学生がプロフィールを登録する。
   - 例：経済学部、サークル、趣味、目標、やりたいことを登録する。

3. AI寮母が学生に地域活動を提案する。
   - 例：地域イベント、ボランティア、店舗の手伝い、文化イベントを提案する。

4. 学生が経験を投稿する。
   - 例：参加したイベントの感想、利用した病院の情報、おすすめ店舗を投稿する。

5. AIが投稿された情報を次の学生に活用する。
   - 例：先輩の投稿をもとに新入居者へ回答する。

6. 保護者向けレポートのイメージを見せる。
   - 例：今月のAI利用状況、頑張ったこと、生活上の大きな問題がないことを簡潔に表示する。

### 12.5 デモで注意すべきこと

- **学生・保護者・先輩の個人情報**を実在のものにしない（架空維持）。
- 実在店舗・実在施設情報を返す際は「情報が古い可能性がある」旨を必ず併記する。
- 医療判断をAIが断定しない。
- 保護者向けレポートが監視に見えないようにする。
- 地域情報が広告っぽくなりすぎないようにする。
- 学び・成長・つながりが伝わる体験にする。
- 生活支援だけで終わらず、行動のきっかけまで見せる。

## 14. 課題・リスク・要検討点

### 14.1 課題の定量データが弱い

書類選考時点では、チーム内アンケートや参考調査を用いているが、決勝に向けてはより説得力のある定量データが必要になる可能性がある。

特に必要なデータは以下である。

- 一人暮らし学生の不安に関するデータ。
- 保護者の不安に関するデータ。
- 学生マンション内の交流の少なさに関するデータ。
- 学生と地域の接点不足に関するデータ。
- 学生が「何かやりたいが何をすればよいかわからない」と感じているデータ。
- 地域側が学生と関わりたいと考えているデータ。

### 14.2 AI回答の信頼性

AIが誤った情報を回答すると、生活上のトラブルにつながる可能性がある。

特に注意が必要な領域は以下である。

- 医療。
- 法律。
- 行政手続き。
- 緊急対応。
- 防犯。
- 契約関連。

AIは断定的に判断するのではなく、公式情報や緊急窓口へ誘導する設計が必要である。

### 14.3 プライバシー・個人情報

学生のプロフィール、会話ログ、近況報告、保護者レポートを扱うため、個人情報保護が重要である。

要確認事項は以下である。

- どこまでパーソナルな情報を収集するか。
- 会話ログをどのように保存するか。
- 保護者へ何を共有してよいか。
- 学生本人の同意をどう取るか。
- 学生同士の情報をどこまで共有してよいか。
- 地域店舗・団体への情報提供範囲。
- データ削除・退会時の扱い。

議論では、これらについて弁護士に確認したいという認識が示されている。

### 14.4 保護者レポートが監視に見えるリスク

保護者向けレポートは価値がある一方で、学生から見ると監視に感じられる可能性がある。

そのため、以下の設計が必要である。

- 学生本人の明確な同意。
- 共有する情報の限定。
- 会話内容そのものを安易に共有しない。
- 学生が共有範囲を管理できる仕組み。
- 「見守り」ではなく「自立支援」として表現すること。

### 14.5 投稿内容の確認・管理コスト

AI寮母は学生や地域からの投稿を扱うため、情報の正確性や安全性を保つ必要がある。

課題は以下である。

- 誰が投稿内容を確認するか。
- 誤情報をどう修正するか。
- 古くなった情報をどう更新するか。
- 悪意ある投稿をどう防ぐか。
- 医療・行政・契約関連の情報をどう扱うか。

### 14.6 学生投稿のインセンティブ設計

学生が継続的に情報投稿するには、インセンティブが必要である。

候補は以下である。

- ポイント付与。
- 地域店舗のクーポン。
- 貢献度の可視化。
- 保護者レポートや就活記録への活用。
- 同じマンションの後輩に役立つという貢献感。

### 14.7 地域店舗が広告費を払う理由

地域店舗から広告収益を得るには、広告費に見合う効果を示す必要がある。

要検討事項は以下である。

- 学生送客数をどう測定するか。
- クーポン利用数をどう示すか。
- 掲載料2,000円/月が妥当か。
- 小規模店舗が支払える価格か。
- 広告ではなく地域貢献・学生応援として訴求する方がよいか。

### 14.8 緊急時対応の責任範囲

緊急時通知や体調不良時の案内を行う場合、責任範囲を明確にする必要がある。

AIが医療判断をするのではなく、適切な相談先、救急窓口、公式情報へ誘導する設計が必要である。

### 14.9 機能過多によるブレ

現時点では、機能が多く、プレゼンや開発で焦点がぼやける可能性がある。

特に、以下のどれを主軸にするかを決める必要がある。

- 保護者の安心。
- 学生の生活支援。
- 学生の学び・成長。
- 地域との接続。
- 学生同士のつながり。
- JSBの物件価値向上。

最新の議論では、「何かやりたいけど何をすればよいかわからない学生に対する、学び・成長・つながりのきっかけづくり」が重要な軸になっている。ただし、保護者の安心も継続して残っている。

## 19. 開発時にAIへ渡す場合の注意

このドキュメントをAIに渡して開発支援を受ける場合は、以下を前提として扱う。

保護者レポート、クーポン、広告管理、学生同士のマッチングは、時間が限られる場合はモックや画面イメージに留める可能性がある。

### 20.3 発表の構成案

発表構成としては、以下の流れが考えられる。

1. 学生マンションには、生活知・地域情報・学生の関心が埋もれている。
2. その結果、学生は一人暮らしに不安を抱え、地域や学生同士とつながれず、成長機会を逃している。
3. 地域側も学生と関わりたいが、接点がない。
4. AI寮母は、学生と地域の情報をAIに蓄積し、暗黙知を形式知化する。
5. 学生は生活相談ができ、地域活動や学びの機会を提案される。
6. 保護者は適度な安心感を得る。
7. JSBは物件差別化と知識資産を得る。
8. 地域社会は学生との接点と送客を得る。
9. 知識共有、ポイント獲得、地域消費、地域活性化の循環が生まれる。
10. 学生マンションは、住む場所から、学び・成長・つながりを生むインフラへ進化する。

## 22. 最終整理

AI寮母は、初めて一人暮らしをする学生の不安を解消する生活支援AIとして出発した。しかし、決勝に向けた議論を通じて、単なる生活支援にとどまらず、学生の学び・成長・つながりのきっかけを生むサービスとして発展している。

このプロダクトの核心は、学生マンションに存在する生活知、地域情報、学生の興味関心、地域側のニーズをAIで形式知化し、学生・保護者・JSB・地域社会に価値を提供する点にある。

今後の最大の課題は、機能が広がりすぎているため、決勝発表とMVP開発に向けて、どの課題を最優先に置くかを明確にすることである。

現時点では、「一人暮らしの不安解消」と「何かやりたいが何をすればよいかわからない学生への成長機会提供」の2軸が特に重要である。

AI寮母は、学生マンションを単なる住居から、学生の経験が蓄積され、地域とつながり、次の学生へ知識が受け継がれる場所へ変える提案である。
